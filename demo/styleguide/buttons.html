Basics

While buttons in Encore-UI can be marked up multiple ways: through button, input type="button" or the rxButton direcitve, which adds dynamic actions to loading indicators and messages - they should be styled using the .button class.

Customizing Buttons

Buttons are sized using xl, lg, sm and xs classes. Not adding the additional class defaults the size to medium.

Using Colors with Buttons

ENCORE-SPECIFIC: In the context of the Encore-UI, when is a button blue and when is a button green?

  • If the form you are submitting results in CREATING or ADDING something, use green.
  • If the form you are submitting is within a modal, use green.
  • If the form you are submitting results in editing something or something auxiliary (like getting metadata from a different sources) keep it blue.
  • If you are providing a button for the user to exit a series of modals (with no submit), use blue.

Using Spinners with Buttons

The Framework provides a spinner style for use with buttons. Note that different spinner styles are used when content is loading or being fetched (see Using Colors and Icons With Links demo). When a user takes an action with results that cannot be immediately displayed, a spinner is shown on a disabled button state until the call completes or the results of the user's action can be displayed in the UI. Text for buttons with spinners should describe the action being taken using "-ing" verbs, so the user understands why the button appearance has changed. Note that the disabled state ensures the user cannot submit the same data multiple times while the initial action completes.

Using Colors and Icons With Links

Colors

Status Classes are used to provide a clearer status regarding a simple message. Within the context of Encore, we use the following colors to convey action links:

  • Green (.msg-action) is for create/add actions. When part of a list, they appear at the top.
  • Blue (.msg-info-blue) is for edit/modify actions. When part of a list, they appear second.
  • Orange (.msg-info) is for build an action upon something. When part of a list, they appear third.
  • Red (.msg-warn) is for destructive actions. When part of a list, they appear last.

HEADS UP: Because these classes are colors, they can also be applied to regular text as well. This can cause confusion between when something is a text and something is an action. This is a design pattern discrepancy that will need to be resolved in a future iteration.

Icons

We've been using the Font Awesome library as a way to primarily visualize actionable items. For the majority of Encore, actionable links are used in two places - in the upper right hand section of a details page, flushed right with the page-actions and actions-area classes, or in the rxActionMenu directive.
